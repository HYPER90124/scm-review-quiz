# 供应链复习答题程序实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 单文件 `index.html` 答题程序，覆盖《供应链复习资料.docx》全部知识点（约 98 题），本地可用并部署到 GitHub Pages。

**Architecture:** 题库以纯 JSON 存放在 `<script id="question-data" type="application/json">` 中，应用逻辑为原生 JS，三个视图（首页/答题/错题本）由同一页面切换，进度存 localStorage。

**Tech Stack:** 原生 HTML/CSS/JS；python 做题库校验；git + gh CLI 部署 GitHub Pages。

## Global Constraints

- 单文件 `index.html`，零外部依赖（无 CDN、无外链字体/图片）。
- 题型：single | multi | judge | essay。
- 星标规则：chapter ∈ {4,6,7,8} 或 资料括号标注题型 或 标注（重要）。
- 手机（375px）与桌面均可用；localStorage 不可用时降级内存模式。
- 题目对象结构见设计文档 `docs/superpowers/specs/2026-07-05-quiz-app-design.md`。

### Task 2: 应用 UI 与逻辑（同一 index.html）

**Files:**
- Modify: `index.html`（补全 CSS + JS）

**Interfaces:**
- Consumes: Task 1 的 question-data JSON。
- Produces: 三视图 SPA——`renderHome() / renderQuiz(list, index) / renderWrongBook()`；localStorage key `scmquiz-v1`，结构 `{version:1, records:{[id]:{status:'right'|'wrong', tries:n}}, wrong:[id...]}`。

**Steps:**

- [ ] **Step 1:** CSS：移动优先，卡片布局 max-width 680px 居中；选项为大号可点按钮；星标金色 ★；题型彩色标签；进度条。
- [ ] **Step 2:** 首页：13 章节卡片（题数+已答对数）、全部顺序刷题、错题本（含数量徽标）、总进度、重置按钮（confirm）。
- [ ] **Step 3:** 答题流程：single 点选即可提交；multi 多选后点"提交"；judge 选对/错；essay 点"查看参考答案"后自评。提交后即时反馈：对→绿色；错→红色并高亮正确答案，显示 reference 解析、judge 的 correction。底部上一题/下一题。
- [ ] **Step 4:** 错题本：答错或 essay 自评"没掌握"入本；在任意模式中答对/自评掌握则移出。
- [ ] **Step 5:** localStorage 读写 + try/catch 降级；版本不符自动重置。
- [ ] **Step 6:** commit。
