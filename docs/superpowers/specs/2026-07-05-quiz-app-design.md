# 供应链管理复习答题程序 — 设计文档

日期：2026-07-05
状态：已获用户批准

## 目标

为《供应链管理》课程期末复习制作一个答题程序，手机和电脑浏览器都能直接使用。
题目全部来自《供应链复习资料.docx》（含 5 张图片中的表格），**资料中所有知识点都必须至少出一题**。

## 技术方案

单文件 `index.html`：题库（JS 数组）、样式、逻辑全部内嵌，零外部依赖。

- 本地交付：桌面"供应链复习"文件夹中的 `index.html`，双击即用。
- 在线交付：推送到 GitHub 仓库并开启 GitHub Pages（安装 gh CLI + 用户登录授权后由 Claude 自动完成）。
- 响应式布局：手机竖屏单列，电脑居中卡片（max-width 限宽）。

## 题库规则

题量约 90 题，覆盖第 1–13 章全部知识点，包括图片表格：

- 表 3-1 效率型 vs 反应型供应链特点比较
- 图 3-6 供应链设计类型与运作模式选择矩阵
- 表 5-1 四种仓配模式的比较
- 表 5-2 云仓与传统仓的区别
- 表 10-1 流通加工与生产加工的区别

**题型分配**：

1. 资料中括号标注了题型的，严格按标注出题（如：牛鞭效应→大题，供应链管理特征/内容→多选，"物流是供应链中的一部分"→判断改错，安全库存→大题等）。
2. 未标注的按知识点性质选最可能考的题型：
   - 提出者/年份/单一事实 → 单选题
   - 特征、分类、内容列举 → 多选题
   - 定义细节可设陷阱 → 判断对错并纠正题
   - 需成段阐述的概念 → 文字大题

**星标 ★ 规则**（三者满足其一即打星）：

1. 来源为第 4、6、7、8 章的题；
2. 资料中用括号标注了题型的题；
3. 资料中标注（重要）的知识点出的题。

## 答题交互

- **单选/多选**：选择后提交，立即判对错；答错时高亮正确答案并显示知识点原文解析。
- **判断对错并纠正**：先选"对/错"，提交后显示答案；若原句错误，给出纠正后的正确表述。
- **文字大题**：先自行回忆，点"查看参考答案"后自评"掌握了 / 没掌握"。
- 答错或自评"没掌握"的题自动进入错题本。

## 功能与页面

1. **首页**：13 个章节卡片（每章题数 + 完成进度）、"全部顺序刷题"入口、"错题本"入口。
2. **答题页**：一题一屏；显示章节名、题型标签、星标 ★；答完即时反馈；可上一题/下一题。
3. **错题本**：收集答错/未掌握的题，可反复重刷；答对或自评掌握后自动移出。
4. **进度保存**：localStorage 保存答题状态与错题本；手机与电脑进度各自独立。

## 数据结构（题目对象）

```js
{
  id: "c2-q8",          // 章节-序号
  chapter: 2,            // 1–13
  type: "essay",         // single | multi | judge | essay
  starred: true,
  question: "…",
  options: ["A…","B…"],  // single/multi 用；judge 固定 对/错
  answer: [0,2],         // single: 索引；multi: 索引数组；judge: true/false
  correction: "…",       // judge 且原句为错时的纠正表述
  reference: "…",        // essay 参考答案；其他题型作为解析
}
```

## 错误处理与边界

- localStorage 不可用（隐私模式）时降级为内存模式，提示进度不会保存。
- 进度数据带版本号，题库更新后不兼容时自动重置。
